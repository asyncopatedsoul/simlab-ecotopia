# `la-greater-v1` region pack

Greater Los Angeles region pack — **dev stub** until bd-spec.2b lands the iNat/GBIF ingest and CC-BY photo acquisition.

## What's here

- `curation.csv` — 25 hand-picked LA species spanning the five MVP scenarios. Real scientific + common names; iNat taxon IDs left blank pending reconciliation in bd-spec.2b.
- `meta.json` — pack metadata (id, version, region bbox).
- `built/la-greater-v1.biotope-region/` — the built pack (regenerated by `npm run region-pack:build`).

The dev pack ships with **stub photos** — sharp-generated 64×64 webps, color-coded by taxon id. Every stub asset's `attribution` reads `"Stub placeholder (dev build) — replace with CC-BY iNat photo before ship"`. The runtime can load them, but they are obviously not real photos.

## Build

```bash
npm run region-pack:build
```

Equivalent to:

```bash
tsx tools/region-pack/build.ts \
  --curation content/region-packs/la-greater-v1/curation.csv \
  --meta     content/region-packs/la-greater-v1/meta.json \
  --out      content/region-packs/la-greater-v1/built
```

The build script runs the §9 validators after writing, asserts the 50 MB budget, and exits non-zero on any error-severity issue.

## Adding species

1. Append a row to `curation.csv` with a fresh `id` (do not renumber existing rows — `id` is a stable referent).
2. Tag it with the scenario(s) that need it.
3. Re-run `npm run region-pack:build`.

## Replacing stubs with real photos

Once bd-spec.2b ships:

```bash
# Acquire CC-BY photos via the iNat API into a directory keyed by taxon id:
#   <photos-dir>/1/001.webp, <photos-dir>/1/002.webp, ...
tsx tools/region-pack/build.ts \
  --curation ... --meta ... --out ... \
  --photos content/region-packs/la-greater-v1/photos
```

When `--photos` is supplied the stub generator is bypassed and real bytes go in.

## Provenance

All taxon names and binomials in `curation.csv` are public-domain taxonomic facts. iNaturalist taxon IDs (when populated) are public references to iNat's taxonomy. Photos must be CC-BY-4.0 (or CC0/CC-BY-SA) with attribution preserved in `photos_manifest.attribution` and `photos_manifest.source_url`. Audio (post-MVP) will follow the same rule via Xeno-canto / Macaulay Library.
